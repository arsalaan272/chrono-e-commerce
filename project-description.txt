The Home page is the landing and main entry point to the e-commerce application. It displays featured products, highlights key product categories through engaging banner sections, and calls users to browse different product categories. It's designed in a way to drive user engagement and conversion through visual storytelling and strategic product placement.

Key Features:

A. Hero Section
A promoted banner on top with striking imagery, along with call-to-action buttons
Built using the Hero component from src/components/Hero.js

Grabs the immediate user attention on landing onto the site.
B. Section of Features Products
Product listing- displays 6 handpicked products from the entire product catalog (first 6 items)
Uses the ProductList component to render product cards in a grid layout
Includes a "View All Products" button that routes to /products

Products featured here include the Apple Watch Series 9, Samsung Galaxy Watch 6, Fitbit Versa 4, iPhone 15 Pro, Samsung Galaxy S24 Ultra, and Google Pixel 8 Pro.

Each product card depicts the following: an image, the name of an item, price in INR ₹, rating using stars, a quick description, and an "Add to Cart" button.

2. CATEGORY PAGES (/category/smart-watches, /category/smart-mobiles, etc.)
What it Does:
Category pages allow users to browse products filtered by specific categories. They are dedicated pages for each product type, which contains a beautiful hero header showing category information, followed by the filtered list of products. Category pages improve discoverability and help users jump to the products they are looking for.

Key Features:
A. Dynamic Category Detection Now, category detection can be implemented in the CategoryPage component by using the useParams hook provided by React Router to extract the category from the URL:
Available categories:

smart-watches - Smart watches and fitness trackers
smart-phones - Smart-phones and mobile devices
laptops - Laptops and computing devices
grocery - Groceries and necessities

B. Atrractive Category Header Banner Each category features a full-width header that includes:
Background Image: HQ photo, different for each category

Breadcrumbs Navigation: "Home" > "Category Name" for easy navigation

Category Title: Large, bold h2 heading, such as "Smart Watches"
Category Description: This is the shorthand description of what will be found in that category.
**All text is white with text-shadow for contrast against background images

Title animates in with opacity: 0 → 1 and y: 20 → 0

Card of products grid layout

E. Responsive Layout
Displaying a description on mobile devices only - seen only on small screens with display: { xs: 'block', md: 'none' }

Description of desktop with increased visibility

Manufacture:
Component Structure:

4. ADMIN DASHBOARD PAGE (/admin/dashboard)
What it Does:
The AdminDashboard is where control is effected by an administrator. It provides a full interface for managing the inventory of products, thus allowing admins to add new products, view all dynamically added products, delete products, and manage product features. It's a full CRUD interface built with extensive form validation and real-time feedback.

Key Features:
A. Authentication Check
On mount, checks if adminAuthenticated flag exists in localStorage
